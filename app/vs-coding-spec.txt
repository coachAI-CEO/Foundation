1. Overview
Vision Studio is Screen 9 of Foundation by Ascension. It occupies a dedicated slot in Foundation's navigation — accessible from the desktop sidebar (camera icon) and the mobile bottom nav "Studio" tab. No other Foundation screen is modified.
A user uploads a photograph of any interior space, describes the desired transformation in plain English, and clicks Calculate. Three parallel AI threads analyze the room's spatial geometry and generate three photorealistic variant renders. Simultaneously, an architectural brief streams into the left chat panel and a live cost estimate populates the right panel.
Clicking Generate Blueprint converts the Vision Studio session directly into a Foundation project: tasks are pre-populated from the execution blueprint, materials are loaded into the shopping list, and the app navigates to the new project's Blueprint screen.

3. State Object
The prototype stores all VS runtime state in a single var VS object. The production Zustand slice mirrors this structure:

var VS = {
  variants:      [],     // VSVariant[] - grows as each thread completes
  activeVariant: 0,      // 0-2, index into variants[]
  hasEstimate:   false,  // gates Generate Blueprint button
  isDragging:    false,  // true while user drags the canvas divider
  genActive:     false,  // prevents double-firing Calculate
  uploadedSrc:   null,   // base64 or blob URL of uploaded photo
  _progressInterval: null // cleanup handle for progress bar setInterval
}

The prefix parameter ("d" for desktop, "m" for mobile) threads through every VS function to address the correct DOM IDs. In production, replace this with a single responsive component that reads window width.

5. Generation Sequence
The vsCalculate(prefix) function is the core orchestrator. The exact timing from the prototype is the production spec:

t=0ms      User clicks Calculate
           VS.genActive = true
           Button text: "Analyzing spatial geometry..."
           Canvas empty state hidden, split view shown
           Upload photo placed into original panel
           Thread dots start animating (staggered +400ms each)
           Progress bar setInterval starts (every 350ms, +random*7, caps at 85%)

t=1200ms   Thread 0 (Variant 1) completes
           Thumbnail 0 spinner -> image
           Render panel loading overlay removed
           VS.variants[0].imageUrl shown in render panel
           canvas-variant: "1 / 3"

t=1800ms   vsGenerateBrief fires (600ms after all variants complete)
           Brief typing indicator shown in chat
           Claude API call: brief prompt with user prompt

t=2100ms   Thread 1 (Variant 2) completes
           Thumbnail 1 spinner -> image
           canvas-variant: "2 / 3"

t=3000ms   Thread 2 (Variant 3) completes
           Thumbnail 2 spinner -> image
           canvas-variant: "3 / 3"
           Progress bar -> 100%, then clears after 1000ms
           canvas-mode: "Comparative View"
           Button text: "Recalculate", re-enabled
           Thread dots: green-br for 2500ms, then back to idle

t=3600ms   vsGenerateBrief fires (600ms after all done)
           Brief blocks rendered in chat

t=5200ms   vsGenerateEstimate fires (2200ms after all done)
           live-dot: gold + pulseDot animation
           live-txt: "Processing..."

t=6600ms   Estimate renders (1400ms after estimate start)
           vsAnimateNum: budget-num 0->total over 1200ms (desktop) / 1000ms (mobile)
           budget-fill: width 0%->68%, transition 1s ease (after 200ms delay)
           est-mat, est-lab: textContent set directly
           mat-list: innerHTML = shopping list items
           live-dot: green-br, live-txt: "Live"
           VS.hasEstimate = true

7. Production API Routes
The prototype uses hardcoded demo data (VS_DEMO_VARIANTS, VS_DEMO_EST, hardcoded brief text). The production implementation replaces these with real API calls.

POST /api/vision-studio/generate
Fires 3 parallel Gemini 2.0 Flash image generation threads. Returns streaming NDJSON — one JSON line per variant as it completes. Client reads the stream and calls vsSelectVariant as each arrives.

// Request
{ imageBase64: string, prompt: string, houseId: string }

// Streamed response (NDJSON, one line per variant)
{ variantIndex: 0|1|2, imageUrl: string, status: "done"|"error" }

// Three creative seeds used for diversity:
// Thread 0: "photorealistic, warm natural materials"
// Thread 1: "photorealistic, cool industrial palette"
// Thread 2: "photorealistic, luxury refined finishes"

POST /api/vision-studio/brief
Calls Claude Sonnet 4 with a structured prompt. Returns three blocks: Architectural Analysis, Material Specification, Execution Blueprint. Response is JSON, rendered into .vs-brief-block HTML by vsGenerateBrief.

// Request
{ prompt: string, imageBase64: string, variantUrl: string }

// Response
{ blocks: Array<{ label: string, items: string[] }> }

POST /api/vision-studio/estimate
Calls Claude Sonnet 4 with the accumulated brief text. Returns a VSEstimate object. Fires silently 2200ms after all variants complete.

// Request
{ briefText: string, prompt: string }

// Response — VSEstimate
{
  total: number,
  materials: number, labor: number, fixtures: number, contingency: number,
  timeline: { prep: string, execution: string, finishing: string },
  materials_list: Array<{ name: string, price: number, category: string }>
}

POST /api/vision-studio/refine
Chat refinement. Receives message + conversation history + current estimate. Returns AI reply and an optional estimate delta. If the response body contains <estimate_update>{...}</estimate_update>, parse and apply to the estimate panel.

// Request
{ message: string, history: VSBriefMessage[], estimate: VSEstimate|null }

// Response
{ reply: string, estimateUpdate?: Partial<VSEstimate> }

10. Blueprint to Project Handoff
When Generate Blueprint is clicked, VS creates a complete Foundation project and navigates to it. This is the key integration point between Vision Studio and the rest of Foundation.

// vsGenerateBlueprint production implementation
async function handleBlueprint(session, estimate) {

  // 1. Extract project name and room from prompt
  const name = extractProjectName(session.prompt)
  const room = extractRoom(session.prompt)

  // 2. Create Foundation project
  const project = await db.insert(projects).values({
    house_id: session.house_id,
    name, room,
    budget: estimate.total,
    status: "planning",
    color: "#d4903a",
  }).returning()

  // 3. Create tasks from execution blueprint steps
  const executionBlock = session.brief.blocks.find(b => b.label === "Execution Blueprint")
  for (const step of executionBlock.items) {
    await db.insert(tasks).values({
      project_id: project.id,
      name: step.replace(/^0d+ -- /, ""),
      tag: "DIY", who: "Both", done: false
    })
  }

  // 4. Create materials from shopping list
  for (const mat of estimate.materials_list) {
    await db.insert(materials).values({
      house_id: session.house_id,
      project_id: project.id,
      name: mat.name, price: mat.price, priority: "needed"
    })
  }

  // 5. Lock VS session
  await db.update(vs_sessions).set({ project_id: project.id, locked: true })
    .where(eq(vs_sessions.id, session.id))

  // 6. Navigate to blueprint
  navigate("blueprint")  // Foundation routing
}
