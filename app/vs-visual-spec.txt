
VISION STUDIO
Visual Design Specification
Foundation by Ascension  |  Screen 9

All values are ground-truth extracted from foundation-with-vs.html

1. Design Philosophy
Vision Studio inherits Foundation's complete design system without introducing a single new token. Every color, font, radius, spacing unit, and animation reuses the existing Foundation CSS custom properties.
Visual direction: precision instrument. The UI reads like calibrated equipment. Tight 2px corner radius on interactive elements (vs Foundation's 16px cards), rule lines at 1px, gold used sparingly as the single interactive accent. Against Foundation's warm obsidian surfaces it feels like a darkroom built inside the app.
Rule: If a value cannot be expressed with an existing Foundation --token, it is wrong. There are no exceptions.

4. Layout Dimensions
Desktop Three-Column Grid
The VS desktop view fills Foundation's d-screen-content area. The three-column grid has these exact widths:

grid-template-columns: 360px 1fr 300px

Left column:   360px  -- Input panel + Architectural Brief chat
Center column: flex-1 -- Lab Canvas (split view + thumbnail strip)
Right column:  300px  -- Live Estimate panel

Column separators: border-right/left 1px solid var(--border)
Column height: 100% of d-screen-content (fills below Foundation topbar)

Canvas Internal Structure
Canvas top area (flex: 1):
  Original panel (flex: 1)  +  1px gold divider  +  Render panel (flex: 1)

Thumbnail strip (flex-shrink: 0):
  height: 80px
  background: var(--card)
  border-top: 1px solid var(--border)
  gap: 9px, padding: 0 18px

Left Panel Internal Structure
Upload + Prompt section (flex-shrink: 0):
  padding: 18px
  border-bottom: 1px solid var(--border)

  Upload zone: min-height 88px
  Prompt textarea: 4 rows (72px min-height)
  Thread dots row: margin-bottom 10px
  Calculate button: width 100%, padding 11px

Brief chat section (flex: 1, overflow-y: auto):
  Section label: padding 12px 18px 0
  Messages area: padding 12px 18px, gap 12px
  Chat input row: padding 12px 18px, border-top 1px solid var(--border)

Right Panel Internal Structure
Header: padding 14px 18px, border-bottom 1px solid var(--border)

Budget display: padding 16px 18px, border-bottom 1px solid var(--border)
  Budget number: Cinzel 32px
  Budget bar: height 2px, margin 10px 0
  Breakdown grid: 2-column, gap 7px

Shopping list: flex: 1, overflow-y: auto
  Section header: padding 10px 18px 6px
  Each item: padding 8px 18px

Blueprint button area: padding 14px 18px, border-top 1px solid var(--border)

There are no intermediate breakpoints. The layout is binary: 3-column desktop (> 768px) or single-column mobile phone shell (<= 768px). This matches Foundation's established pattern.

7. Visual Consistency Checklist
Before shipping, verify all items on this list. Failing any item is a visual regression.

Tokens
	•	All borders use var(--border) or var(--border-warm) only — zero hardcoded rgba values in VS components
	•	All text colors use var(--white), var(--muted), var(--soft) — no hardcoded hex text colors
	•	All gold is var(--gold) or var(--gold-br) — no hardcoded #d4b978 or #ecd49a
	•	All radius is var(--r) or var(--r-sm) — no custom 4px, 8px, or 12px values
	•	No new CSS custom properties introduced. VS uses zero variables not already in Foundation :root

Layout
	•	Desktop three columns: exactly 360px | flex-1 | 300px — not 350/320 or any other value
	•	Thumbnail strip: exactly 80px height — not 72px or 88px
	•	Thumbnail size: exactly 58px tall, aspect-ratio 16/10 — not 60px or 16/9
	•	Canvas divider: exactly 1px wide — not 2px or a border
	•	Divider handle: exactly 18x36px — not 20x40px

Components
	•	Upload zone dashed border — border-style must be dashed, not solid
	•	Active thumbnail: border-color var(--gold) — no box-shadow, no glow effect
	•	Calculate button disabled: bg var(--raised), not var(--card) or var(--lift)
	•	Progress bar fill: var(--gold), track: var(--raised) — not var(--border)
	•	Budget bar fill: var(--amber) — not var(--gold) — these are different colors
	•	Blueprint button hover fills from bottom via ::before translateY — not fade, not background transition
	•	Fullscreen uses backdrop-filter blur(8px) — not a solid or gradient background

Typography
	•	Cinzel in exactly three places: VS panel header, budget number, blueprint button text
	•	DM Sans everywhere else in VS — no Cinzel on labels, chat, estimates, metadata
	•	Section labels: 9px minimum — never 8px for labels with text, only for metadata badges
	•	Brief block labels: exactly 8px, letter-spacing 0.18em, uppercase, var(--gold)

Animation
	•	Chat messages enter with fadeUp 0.22s — not just opacity fade
	•	Fullscreen has backdrop-filter blur — test in Chrome, it must blur the content behind
	•	Thread dots animate in staggered order (0 -> 1 -> 2) not simultaneously
	•	Budget number animates from zero — does not snap to final value on render
	•	Scrollbars: scrollbar-width thin, scrollbar-color rgba(212,185,120,0.18) transparent on all VS overflow panels
